Why harness engineering?

Research shows that the wrapper around an LLM matters as much as the model itself. When Princeton researchers redesigned the agent-computer interface using the same GPT-4 model, performance on SWE-bench roughly tripled — from 3.8% to 12.5%.

The model didn't change. The harness did.

"The harness is what makes the agent run."

— Hugging Face, "Harness, Scaffold, and the AI Agent Terms Worth Getting Right"

What the tool generates

my-project/ ├── AGENTS.md # All AI tools read this ├── CLAUDE.md # Claude Code specific ├── .cursor/rules/ # Cursor rules ├── [lint config] # biome.json / ruff.toml / ... ├── [project config] # package.json / pyproject.toml / ... ├── .husky/pre-commit # Automatically checks on commit ├── .github/workflows/ci.yml # CI pipeline ├── scripts/setup-hooks.sh # One-shot hook setup ├── docs/ │ └── architecture.md # Module boundaries + ADRs ├── src/ └── tests/ # Smoke test, passing out of box

Amber-highlighted files are harness-engineering specific — the feed-forward guides and feedback sensors that make AI coding agents reliable, not just fast.

Based on Martin Fowler's harness engineering framework: guides tell the agent what to do before it acts; sensors catch problems after it acts and let it self-correct. You need both.

How it works

1. Run one command

npx @harness-ready/create-app in your terminal.

2. Answer a few questions

Choose your language, framework, AI coding agent, team mode, and CI provider.

3. Get a harness-ready project

Install dependencies, activate hooks, and start building — your AI agent already knows the project.
